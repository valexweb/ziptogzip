# Инструкция по использованию

Консольная утилита `ZipToGzipConverter` предназначена для преобразования однофайловых ZIP-архивов в формат GZIP без повторного сжатия данных.

---

## Описание работы

Программа выполняет прямой переноса DEFLATE-потока из формата ZIP в формат GZIP. Это позволяет быстро конвертировать архив без затрат ресурсов процессора на распаковку и повторное сжатие.

В процессе выполнения утилита производит строгое валидирование структуры файла:
* Проверку сигнатур Local Header, Central Directory и EOCD.
* Проверку целостности границ данных (защита от выходов за пределы массива).
* Проверку соответствия метаданных между заголовками (CRC, размеры, имя файла).
* Проверку ограничений: архив должен содержать **ровно один файл**, не использовать шифрование, Data Descriptor или версии выше 2.0.

---

## Системные требования

* **JRE (Java Runtime Environment)** или **JDK** версии 8 или выше.
* Запуск из командной строки (Terminal, PowerShell, CMD).

---

## Запуск программы

### Запуск с помощью готового JAR-файла

Синтаксис команды:

```bash
java -jar ZipToGzipConverter.jar <путь_к_исходному_zip> <путь_к_целевому_gzip>
```

#### Примеры:

* **Linux / macOS:**
  ```bash
  java -jar ZipToGzipConverter.jar /path/to/source.zip /path/to/output.gz
  ```

* **Windows:**
  ```cmd
  java -jar ZipToGzipConverter.jar C:\data\source.zip C:\data\output.gz
  ```

---

### Прямой запуск из исходного кода (Java 11+)

Если вы не собирали JAR-файл, но установлен JDK 11+, исходный файл `.java` можно запустить напрямую:

```bash
java ZipToGzipConverter.java input.zip output.gz
```

---

## Обработка ошибок и коды возврата

Утилита выводит понятные сообщения об ошибках в консоль в следующих случаях:

1. **Неверное количество аргументов:**
   Если при запуске не переданы пути к файлам, программа выведет справку по использованию.

2. **Неподдерживаемый формат архива:**
   * Несколько файлов или папки в архиве.
   * Использование Data Descriptor (флаг `0x08`).
   * Использование шифрования (флаг `0x01`).
   * Метод сжатия, отличный от DEFLATE (метод 8).
   * ZIP64 или версии спецификации выше 2.0.

3. **Повреждение файла или структуры:**
   * Отсутствие или повреждение структур EOCD / Central Directory.
   * Несовпадение CRC32 или размеров сжатых/несжатых данных между заголовками.
   * Выход значений полей за фактические границы файла.


# Инструкция по сборке

## Требования

* **JDK (Java Development Kit)** версии **8** или выше.
* Проверить установку в консоли:
  ```bash
  javac -version
  java -version
  ```

---

## Сборка JAR-файла

### Вариант 1. Автоматическая сборка (Java 21+)

Самый простой вариант, если у вас установлена свежая версия Java:

1. Скопируйте исходный код в файл `ZipToGzipConverter.java`.
2. Скомпилируйте код и упакуйте в JAR одной командой:
   ```bash
   javac ZipToGzipConverter.java
   jar cfe ZipToGzipConverter.jar ZipToGzipConverter ZipToGzipConverter.class
   ```

---

### Вариант 2. Классическая сборка (Java 8–20)

1. **Скомпилируйте исходный файл:**
   ```bash
   javac ZipToGzipConverter.java
   ```
2. **Упакуйте проект в JAR:**
   ```bash
   jar cfm ZipToGzipConverter.jar manifest.txt ZipToGzipConverter.class
   ```

После успешной сборки в вашей папке появится файл `ZipToGzipConverter.jar`.
